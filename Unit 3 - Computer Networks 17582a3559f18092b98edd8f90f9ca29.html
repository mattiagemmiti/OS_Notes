3.1. Principles of Data Transmission

Communication

Human communication has evolved from direct speech and gestures to long-distance methods like smoke signals, mail, telegraphs, and analog phones. Today, most communication is digital, including how computers exchange information. However, even digital communication relies on analog signal transmission, where digital data is mapped to physical quantities like current, voltage, light pulses, or electromagnetic waves. Common transmission medium include copper cables, fiber optics, and radio waves.

  • Digital vs. Analog Transmission
    • Digital transmission: Deals with binary data (0s and 1s), also called symbols.
    • Analog signals: The actual transmission is in the form of physical signals like voltage, light, or radio waves.
  • Transmission Medium is a carrier for the propagation of the signal changes in a physical quantity.
    • Copper cables: Traditional wiring, prone to interference but widely used in local networks (LAN), for current/voltage or electromagnetic waves.
    • Fiber optics: High-speed, light-based transmission, ideal for long-distance and high-volume data.
    • Radio waves: Wireless transmission, used in Wi-Fi, cellular networks, and satellite communication.
  • Data trasmission Coding

    In data transmission, digital values (e.g., "0" or "1") are mapped to physical quantities in the transmission medium, such as voltage levels. However, this mapping is prone to distortion from interference during transmission. To ensure accuracy, coding defines acceptable signal intervals and includes mechanisms to detect and correct errors.

    Key coding steps in data transmission:

    1. Line Coding: Maps digital values to physical signals (and viceversa), allowing reconstruction on the receiver side.
    1. Channel Encoding (or error correction): Adds checksums (a redudancy correction) to the data to be transmitted. This helps us spot if something goes wrong during transmission. Sometimes, it even lets us fix errors automatically.
      1. Checksum: a number derived from a set of other numbers (e.g., through summation) used to detect errors by comparing it to a previously derived number over the same set.
    1. Source Encoding: Compresses original data before transmission, to reduce the burden on channels, often resulting in "lossy" content, where multimedia quality decreases to match available bandwidth. While source encoding is less significant in PC communication, it plays a critical role in transmitting multimedia over low-capacity connections like mobile networks, preventing interruptions by adapting the data stream to the available data rate.
      1. Channel capacity : It is the maximum data rate that a transmission channel can provide, and it is measured in bits per second (bps).
Line codes and modulation

Line coding process is fundamental in data transmission, as it allows computers to communicate over various physical channels like copper cables, fiber optics, or radio waves.

  • Baseband Transmission:

    Uses signals without a higher-frequency carrier. It transmits signals in their original frequency range. Example: Amplitude modulation, where +/- 5 volts represent "0" and "1" in transmissions over copper cables.

  • Passband Transmission:

    Shifts baseband signals to a higher frequency range using a carrier signal.

  • Manchester Code:

    Manchester encoding is a phase encoding method where voltage rising (negative to positive) = binary "1" and voltage falling (positive to negative) = binary "0."

    The voltage level changes in every signal period (i.e., at every binary value), ensuring self-clocking. This allows the sender and receiver to synchronize implicitly without needing a separate clock signal, as the transitions help in the clock recovery scheme.

    In Manchester encoding, the sender adjusts the voltage at twice the data rate frequency, ensuring that each bit period contains a transition. This design allows the receiver to recover the sender's clock frequency and maintain synchronization throughout the data transmission (clock recovery).

Modulation: Converts digital data into analog signals for transmission over long distances (e.g., AM, FM, QAM).

Error Detection Mechanisms
  • Challenges:

    During transmission, signals can suffer from distortions like noise or attenuation despite synchronization. Mechanisms like error detection (e.g., checksums or parity checks) and retransmission ensure data accuracy even if signal quality degrades during transmission.

  • Parity Check

    A digital method for error detection (and, if possible, automatic error correction) based on the evaluation of the parity bits, which supplement the payload bits until they are either an even number (even parity) or an odd number (odd parity).

    • Binary data is divided into blocks of n bits, with an additional parity bit appended, that acts as a checksum.
    • How it works: The binary data are packed into blocks of n bits each, appended by a parity bit that acts as a checksum. If the number of bits that are set to “1” within a block is odd, then the parity bit is also set to “1” before being sent over the line; otherwise, it is set to “0.” The receiver can then check whether this rule has been
      observed for each received block—if the total number of “1” symbols in a block is now (including the parity bit) uneven, a transmission error must have occurred.
    • Enhanced Error detection: After transmitting m data blocks, an additional parity block is sent, consisting exclusively of parity bits in each bit position (i.e., for the columns) of the previous m blocks, using row and column parity checks, the receiver can pinpoint the exact corrupted bit and often times correct it automatically
    • Example

      If the bit in question (marked by the lightning bolt symbol) has been corrupted and thus
      transmitted incorrectly, the receiver can detect from the parity bit that there must be an error somewhere in the block, as the total number of “1” symbols therein is not even. Furthermore
      (in this example), after every m = 4 blocks, there is another block that contains no payload data. Rather, it contains only parity bits for the columns of the last four blocks. And, again, the receiver can detect that there must have been a transmission error, this time in the third column of one of the four blocks. With these two coordinates (third column and third block), it becomes clear exactly which bit has been corrupted, and a correction (reverting the wrongly received “1” back to a “0”) is possible.

Operating Modes and Multiplexing

Data communication modes and methods vary based on directionality (from sender to receiver and from receiver to sender) and the ability to handle multiple data streams.

Communication Modes:
  1. Simplex: Data flows only in one direction. Common in systems like Digital Audio Broadcasting (DAB), but rarely used in computer networks.
  1. Half-Duplex: Data flows in both directions, but only one side transmits at a time. Similar to walkie-talkie communication and still used in Wi-Fi networks, where only one station transmits at a time to avoid interference. Active use of the communication medium is handed over to the other side, either explicitly (e.g., through keywords like “over”) or implicitly (by waiting until the channel is free).
  1. Full-Duplex:Data flows simultaneously in both directions but it requires separate channels for each direction. Used in telephone conversation, where both parties can speak and listen at the same time
  1. Multiplexing: Allows multiple senders and receivers to communicate simultaneously by dividing time, space, frequency, or wavelengths:
    1. Time Division Multiplexing (TDM): Divides channel capacity into time slices for multiple data streams in a quasi-parallel.
    1. Space Division Multiplexing (SDM): Uses parallel lines to transmit signals.
    1. Frequency Division Multiplexing (FDM): Uses different carrier frequencies to transmit signals simultaneously.
    1. Wavelength Division Multiplexing (WDM): Uses different laser colors for optical fibers.

    Multiplexing methods can be combined with each other and with simplex, half-duplex, or full-duplex modes, providing complex and flexible options for managing digital data streams.

3.2. Network Topologies

Topology: The topology of a network describes the basic structure of the wiring/ cabling to which the various devices are connected.

  • Bus Topology:

    A bus topology connects multiple devices using a single communication medium (that can comprise several parallel lines or transmission frequencies) but, for the topology, only one device can send data at a time, while another receives it ( simplex or half-duplex communication). Challenges arise in managing simultaneous communication through media access control protocols.

    • Advantages: Easy to set up and requires minimal cabling.
    • Disadvantages:
      • If the bus fails, the entire network stops working.
      • Performance worsens as more devices are added.
    • Modern Use: Rarely used due to inefficiency for wired communication between computers, replaced by switched networks.
  • Switches

    Switches are network components that efficiently manage and control data exchange between multiple devices.

    • How They Work: A switch dynamically connects devices, allowing up to n/2 simultaneous device pairs (if n devices are connected). They can either cross or straight connect inputs with two outputs.
    • Internal Switching Mechanisms:
      1. Crossbar Switches: Highly flexible and can connect many devices at once. However, they are expensive due to their internal architecture because they use a grid of switching elements (n² elements for n devices).
      1. Permutation Switching Networks: Less expensive but less flexible than crossbar switches. They can connect many communication- partners, but not all combinations simultaneously.
    • Switch vs. Hub:
      • Hubs broadcast data to all devices like a shared bus (all connected devices receive all data) , leading to collisions and reduced performance. It’s is a switch in its simplest form but functions like a shared bus.
      • Switches forward data only to the intended recipient, enabling faster, collision-free communication.
    • Packet Switching

      Data streams are broken into smaller data packets of a standard size, each containing additional routing information (meta-information) that the intermediate network nodes (e.g., hubs or switches) evaluate in order to forward the packets to the appropriate destination., forming the backbone of modern internet communication. .

    • Significance: Packet switching is fundamental to modern internet communication, improving efficiency and reliability.
  • Star and Tree Topologies:

    Star Topology: Devices connect to a central switch, enabling direct communication.

    Tree Topology: Multiple stars link together through switches, forming a hierarchy.

    • Structured Cabling:
      • Tertiary or Horizontal Cabling: Connects devices to a room switch.
      • Secondary or Vertical Cabling: Links floor switches to a central switch.
      • Primary or Site Backbone: Connects switches across buildings.

    Advantages:

    • Easy to set up and troubleshoot.
    • Failure of a single cable does not affect the entire network.

    Disadvantages:

    • If the central switch fails, the entire network goes down.
  • Mesh Topologies:

    Mesh networks offer high reliability by creating multiple communication routes. Redundant communication routes enhance fail-safety and allow dynamic path selection.

    Advantages:

    • High fault tolerance: Data can take another path. if a connection fails.
    • Suitable for wide area networks (WANs).

    Disadvantages:

    • Expensive to set up because it requires lots of cabling.
    • Complex to manage.

    Types of Mesh Topology:

    1. Fully Meshed Topology: Each node is directly connected to every other node. Advantages: High reliability, as connections provide redundancy; any node can communicate directly with another.
    1. Meshed Topology: Nodes are interconnected with multiple paths, but not every node is directly connected. Advantages: Provides fail-safety through redundant paths and dynamic path selection, making it robust against node failures.

Size and Extent of Networks

For the hierarchy of networks, the boundaries between these levels are flexible.

  1. Network on a Chip (NoC): Smallest scale network connecting components within a single microchip.
  1. Chip Area Network (CAN): Connects microchips directly, such as the CPU and main memory.
  1. Local Area Network (LAN): A network within a limited area (e.g., offices or schools). Devices at the same location communicate with each other.
  1. Metropolitan Area Network (MAN): Connects multiple LANs over a larger area like a city.
  1. Wide Area Network (WAN): A large-scale network (e.g., a country), built with a developed mesh topology for resilience.
  1. Global Area Network (GAN): Worldwide networks connected through submarine cables or satellites, supporting global communication .

3.3. The OSI Reference Model

The Open Systems Interconnection (OSI) reference model, developed by the International Organization for Standardization (ISO) in the 1980s, offers a conceptual framework for understanding how computer networks communicate. It is made up of seven layers, each responsible for a specific role in the data communication process.

While the OSI model is not strictly followed in real-world implementations and is not a mandatory standard for network device manufacturers, it has significantly improved network compatibility, efficiency, and communication.

Each layer interacts with the ones directly above and below it through well-defined interfaces, and communication between systems (like clients and servers) is handled through protocols operating within the corresponding layers.

  • Although not officially part of the original OSI model, the human user is sometimes considered an eighth layer.

  • Layer 1: Physical Layer

    This layer focuses on the physical transmission of raw data between devices using hardware like cables, connectors, and signals. Early networks used coaxial cables, but today, Ethernet cables with RJ45 connectors are more common. These cables often have twisted pairs of wires to reduce interference. Higher layers rely on this layer for all data transfer.

    • In high-speed networks, optical fiber cables are used.
      • Multi-mode fibers are cost-effective but prone to signal distortion over long distances due to their thicker core.
      • Single-mode fibers are thinner, expensive, and ideal for long-distance communication due to their straight light path.
    • Example of cable connector types
  • Layer 2: Data Link Layer

    This layer ensures data can move correctly between devices in the same network. It includes:

    Error Detection: Detects and corrects errors in data transmission.

    Protocols that control the access of senders to a transmission channel:

    • MAC (Medium Access Control) Protocol: Manages how devices share and access the communication medium in a network. Each device is assigned a unique MAC address (Media Access Control address), hard-coded into the device’s network interface card (NIC). A MAC address is a globally unique identifier, made up of two parts:
      1. A manufacturer identifier (assigned by the IEEE)
      1. A unique serial number (assigned by the manufacturer)
      • In Ethernet and Wi-Fi networks, both parts are 3 bytes long, making the full MAC address 6 bytes (48 bits) in total.
      • MAC addresses are usually written in hexadecimal format, separated by hyphens.

        Example: 12-34-56-78-9A-BC

      • The address ff-ff-ff-ff-ff-ff-ff is reserved as the broadcast address.
  • Layer 3: Network Layer

    This layer decides how data travels between different networks through routing.

    Routers manage this process by connecting different networks and using logical addresses like IP addresses to find the best path for data packets.

    In more distant subnets, routers may be directly connected to one another—such as over wide area networks (WANs)—forming a mesh topology. Routing through these topologies is managed by routing algorithms and protocols, which are part of the network layer.

    • Types of Switches:
      • Layer 2 Switches: Operate at the data link layer. Use MAC addresses to forward data within the same network. More efficient than hubs, as they prevent unnecessary data transmission.
      • Layer 3 Switches: Operate at the network layer. Can route data between different network segments using IP addresses. Often called routers due to their ability to handle routing tasks alongside switching. They can switch between different data link domains (Within a data link domain, devices can communicate directly via their MAC addresses, and hardware-based broadcasting is possible within the same domain).
  • Layer 4: Transport Layer

    This layer ensures data is delivered reliably by splitting data into smaller packets for transmission. Therefore protocols on the fourth layer must ensure that, if expected packets do not arrive, an automatic retransmit of the lost data is performed. For this reason, this type of transport protocol is also called an automatic repeat request (ARQ).

    • If additional data (e.g., parity bits) are added to the packets, they become the headers or trailers. They commonly contain information about the sender and the receiver (e.g., for routing), as well as information about the data packet itself (e.g., its size or parity bits).
    • Protocols: Transmission Control Protocol (TCP) and User Datagram Protocol (UDP) are key protocols operating at this layer.
    • Automatic Repeat reQuest (ARQ)

      A protocol for error correction in data transmission. It works with timeouts (an error that occurs or is automatically generated if an operation and/or the arrival of a response takes too long and thus exceeds a given time period) and with acknowledgments (ACK) and/or repeat requests (negative acknowledgments - NACK).

      1. Stop-and-Wait ARQ:
        • Wait for ACK for each packet from the receiver after each data packet has been delivered before the next one can be sent.
        • If no ACK (NACK) is received within a set time (timeout), the sender resends the last packet.
        • Drawback: low data throughput—even in error-free cases, the sender must still wait for the acknowledgement.
      1. Go-Back-N ARQ:
        1. send up to m data packets, then wait for ACK .
        1. If a packet is missing or faulty on the receiving side, a NACK is sent back for the sequence number N of this data packet.
        1. The sender then retransmits all packets starting from this number, because it is assumed that some of the successive packets already transmitted will have likely also been corrupt and/or lost.
        1. It results in improved data throughput.
  • Layer 5: Session Layer

    This layer establishes and manages the communication sessions between devices, used in fourth layer. It provides the functions of authentication, sychronization, negotiation, and mutual agreement with which communication partners can agree on network parameters in order to establish and maintain the best possible connection, which is called session.

    The session manages and maintains logical connections between devices, ensuring smooth and stateful communication. In network programming, sockets represent these connections, allowing two devices to exchange data bi-directionally.

    Beyond simple connections, the session layer supports group communication, like online meetings, by coordinating multiple participants.

    It also handles errors, terminates sessions gracefully, and releases resources when no longer needed, making it vital for efficient and organized networking.

  • Layer 6: Presentation Layer

    This layer occupies of:

    • Data formatting: translates data between network and application formats into a format that the application layer can accept. For this purpose, the participants either negotiate a common presentation form for the data via the session layer, or the data is converted into a standardized format for the exchange (ex. ASCII for text).
      • One common exchange format is the extensible markup language (XML), which can be used to convert any application data structures into simple text files, which is readable by both humans and machines. After sending such a file, the recipient can reconstruct these data structures with an XML parser and convert them into the respective local format.
    • Compression: Reduces file size for faster transmission. Example: JPEG compression for images or MP3 for audio.
    • Encryption: Secures data for transmission, such as TLS encrypting web traffic.
  • Layer 7: Application Layer

    This layer provides the interface for end-user applications to access network services. Applications (e.g., web browsers or email clients) are not directly housed in this layer, as they are not covered by the OSI model. However, they do interact directly with this layer and make use of its protocol interfaces such as:

    • HTTP/HTTPS: For web browsing.
    • SMTP/IMAP: For email communication.
    • FTP: For file transfers.
Communication within each layer

Communication within the OSI model layers involves a structured interaction where processes or instances at each layer communicate within their respective layer using specific protocols. These protocols also interact with neighboring layers through interfaces to ensure seamless data transfer across the entire network stack.

At the top, applications interact using the application layer's protocols, which rely on the presentation layer to coordinate data formatting and interpretation. These interactions depend on sessions managed by the session layer, which ensures proper synchronization. The transport layer manages data delivery by using logical data packets, which are routed between networks via the network layer, utilizing logical addresses. Within subnetworks, devices communicate directly using hardware addresses, with data transmitted as signals over a physical medium like cables or switches.

This entire process, illustrated in protocol stacks, enables two remote participants to exchange data efficiently. Each layer utilizes the functions provided by the layer below it while offering additional functionalities to the layer above, ensuring reliable and structured communication.

Example:

  1. Layer 7 (Application) on Device A sends a message.
  1. The message moves down through Layers 6 to 1, where it’s prepared for physical transmission.
  1. On Device B, the message moves up from Layer 1 to Layer 7, where it’s understood by the receiving application.

This layered approach ensures that each part of the communication process is handled efficiently and systematically.

Importance of the OSI Model:
  • Interoperability: Provides a standard framework for different systems and networks to communicate.
  • Modular design: Each layer operates independently, enabling easier troubleshooting and upgrades.
  • Protocol Standards: Each layer has specific protocols ensuring network tasks are accomplished correctly.
