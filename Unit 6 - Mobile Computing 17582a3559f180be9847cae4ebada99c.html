Unit 6 - Mobile Computing

Mobile devices operate in various environments, encountering different communication delays, data rates, and costs influenced by location and network conditions. These devices rely on radio networks, which are widely available through modern mobile networks and local Wi-Fi-based computer networks. The seamless integration of mobile devices with the internet is known as mobile computing.

However, mobile computing faces significant challenges due to the unreliable nature of radio network connections (that alone lead to increased and often highly flctating proneness to errors) and the diversity of network types. Movement between network cells (each managed by different ba se stations) often leads to changes in address ranges and internet addresses, causing communication interruptions or performance degradation.

6.1. Basics, Techniques, and Special Features

Basics

Wireless Networks: Mobile devices rely on wireless networks (e.g., Wi-Fi, cellular) that vary in signal strength and data rates based on factors like distance from the base station or the number of users connected.

Communication Issues: when a mobile computer is moved through different cells of a radio network with several base stations, the uninterrupted transfer of responsibility between the different transmitters and stations could present a problem. There are 2 types of interrupted connectivity issues:

  • Unplanned Interruptions: Caused by transmission interference on the radio link or technical issues.
  • Planned Disconnection: User-initiated, such as temporarily disconnecting to save energy or costs, or the device may have approached radio network’s boundaries.

Special Handling in Mobile Systems: Mobility requires constant reconfiguration as devices move between different networks with different IP ranges, causing dynamic network changes that must be handled by protocols designed for mobile computing.

  1. Transport Protocols: Protocols like TCP can ensure reliable data transmission during short interruptions by maintaining transparency and retransmitting lost packets, as long as missing packets on the server side do not cause a timeout.
  1. Proxies: A proxy serves as an intermediary between a mobile device and a server, with special services and protocols, ensuring smooth communication even when the mobile device disconnects. It takes over communication during longer interruptions, performing its role transparently so applications remain unaware of its presence.
The Proxy Approach

The proxy approach is essential for ensuring connection transparency, caching data, and supporting seamless mobile computing transitions, especially during planned and unplanned disconnections.

The proxy facilitates communication on both sides:

  • For the server: It acts as the client, continuing to request or receive data from the server.
  • For the mobile device: It acts as the server, storing data until the device reconnects.
  • Types of Proxies

    There are various proxy types, which are placed at various points on the communication path between client and server:

    1. Proxy Near the Base Station:

    Functions as a client for the server during mobile device disconnections. It maintains the server connection, tolerating long interruptions on the client side. When a mobile device disconnects, the proxy continues to receive and cache data (e.g., downloads) from the server. This data is delivered to the device when it reconnects.

    • A Proxy Close to the Base Station of the Radio Link

    2. Proxy on the Mobile Device

    During disconnections, the proxy emulates a server connection, forming a self-contained "island" with the mobile device. It caches outgoing data (e.g., uploads) from the mobile device. Once the connection is restored, the proxy automatically transfers the data to the server. The mobile device remains unaware of the delayed transfer (Transparent Process).

    • A Proxy Close to (or Directly on) the Client

    3. Combined Proxies

    Uses two proxies simultaneously: one near the base station and one on the mobile device. This approach ensures uninterrupted communication and transparency for both the server and the mobile device. A common Use Case is for Content Delivery Networks (CDNs), such as Amazon Cloudflare and Netify, for efficient and seamless data delivery.

    • A Proxy Close to the Client and Another Close to the Base Station

Use of Contextual Information

Mobile devices use contextual information such as location, environmental conditions, and device status to deliver personalized, context-aware services. This capability enriches applications, improves user interaction, and adapts functionality to suit dynamic environments.

  • Location Awareness:

    Mobile devices change location both in terms of network access and the user’s physical position. Devices can use this information as an input parameter for certain applications, such as a weather app that can display the local forecast or a navigation app that can suggest nearby places of interest.

    The location can be determined either explicitly or implicitly:

    • Explicit Methods: Use of GPS receivers, a worldwide, satellite-supported system for the positioning of electronic devices, to determine precise geographic coordinates.
    • Implicit Methods: Based on properties of the radio network, such as Wi-Fi positioning systems (WPS), which estimate location using Wi-Fi cell data through MAC address of the transmitter or SSID of the wireless network. Wi-Fi-based positioning is often much less accurate than GPS-based positioning, as it can only provide an approximate location.
  • Enriching Applications with Context

    Even when location isn’t essential (e.g., for general websites), contextual data can enhance user experience by providing insights about the user environment:

    • Location: Local time or cultural information.
    • Sensors Usage: Mobile devices can measure brightness, temperature, noise levels…
    • Device-Specific Data: Technical parameters like battery level, usage frequency, or network strength

    Context-aware applications can adapt their behavior, appearance, and content presentation based on these parameters.

6.2. Communication Networks for Mobile Computing

Radio networks are the backbone of mobile computing, enabling mobile devices like smartphones and laptops to move seamlessly across various internet sub-networks. These networks include digital cellular networks (e.g., used for telephony) and local wireless computer networks (e.g., Wi-Fi).

Modulation and Multiplexing for Radio Links

Radio networks transmit digital data over free space using electromagnetic waves. These waves act as a transmission medium. The modulation is a process of mapping digital values (e.g., 0s and 1s) to physical quantities like voltage or electromagnetic waves for transmission. Modulation influences how these digital signals are represented during communication.

Common methods for translating data in binary are:

  • Frequency Shift Keying (FSK): Data is represented by variations in the crests of wave frequency (e.g., higher frequency = "1," lower frequency = "0").
  • Amplitude Shift Keying (ASK): Data is represented by variations of the height in crests in wave amplitude (e.g., high crest = "1," low crest = "0").
  • Phase Shift Keying (PSK): Data is represented by changes in the phase (also called shift or angle) of a wave (e.g., positive shift = "1," negative shift = "0").
  • Multiplexing

    Multiplexing allows multiple transmitters or devices to share the same medium by dividing channel capacity into slices of time, frequency, or space, and by ensuring there is no interference. This problem is avoided automatically by using multiplexing techniques:

    1. Frequency Division Multiple Access (FDMA): Devices transmit simultaneously using different frequencies. This prevents interference by assigning distinct frequencies to each device.
    1. Space Division Multiple Access (SDMA): If two transmitters are located far enough they can transmit on the same frequency without interfering with each other. This technique divides the network area into cells, each with its own base station. Neighboring cells use different frequencies, while distant cells can reuse frequencies.
    1. Time Division Multiple Access (TDMA): Devices share the same frequency by transmitting in assigned time slots. Each device is always assigned a specific time slot per cell. During this period, it is allowed to communicate with the base station. This method can also be combined with FDMA for efficient communication.
Wireless and Mobile Networks

Both Wi-Fi networks and cellular mobile networks can use a combination of FDMA, SDMA, and TDMA to supply mobile computers and devices with internet access.

  1. Wi-Fi Networks

    One should make sure that the station does not transmit on a frequency channel that is used by a neighbor. Modern Wi-Fi devices are capable of automatically complying with this form of space multiplexing. However, devices in the same network share frequencies through time multiplexing.

  1. Cellular Networks

    Earlier generations (e.g., 2G) focused on real-time voice transmission with low data rates (~64 kbit/s). By contrast, modern networks like 4G LTE+ offer high-speed, universal packet-oriented internet connectivity, support downstream data transfer rates of several hundred megabits per second (e.g., for multimedia), and use protocols like Voice-over-LTE (VoLTE) for voice calls.

6.3. Mobile Internet: Protocols and Applications

Mobile Internet Protocols

Most mobile internet protocols are based on the TCP/IP protocol family, but they are often adapted to meet the unique needs of mobile networks. key examples are:

  • Dynamic Address Allocation (DHCP)

    DHCP is widely used protocol for both internet and mobile networks and is particularly effective in local wireless environments.

    • Temporary IP Address Assignment: A DHCP server dynamically assigns an IP address from the local range to a new device when it enters the respective subnet. These addresses can be private and not globally valid. The assigned IP address is valid for a limited time (lease period). Once this time expires, the device must request an extension to keep using the address. If the device leaves the network without renewing the lease, the IP address is freed and made available for other devices.

    This dynamic allocation ensures flexibility and seamless transitions between networks. However, because the IP addresses are temporary and not globally valid, devices cannot act as publicly accessible servers or establish direct client-to-client connections. This limitation aligns with the reduced computing power of mobile devices, which prioritizes mobility over server functionality.

  • Mobile IP Protocol

    A standardized, TCP/IP-based protcol, that addresses the limitations of DHCP by providing a solution for devices to maintain a fixed IP address even as they move between networks.. It provides two IPs for a device:

    • A well-known and already configured permanent address assigned by the home network.
    • A temporary address (called a "care-of address") assigned by the foreign network, which must be globally valid (not private). This address in managed and assigned by a foreign agent, similar to DHCP server.

    Home Agent and Communication: This protocol uses a home agent in the home network to track the device's location and redirect communication when it is in a foreign network. When another device (third-party computer) wants to communicate with the mobile device, it contacts the home agent using the permanent address. If the mobile device is in a foreign network, the home agent intercepts the request, encapsulates the corresponding IP packet (and the subsequent packeta) into an agent-to-agent message (a process called tunneling), and sends it to the foreign agent in the temporary network. The foreign agent then unpacks the message and delivers it to the mobile device via the IP protocol. From the client’s point of view, the packet appears to have been sent by the original computer. Reply packets from the mobile device are sent directly to the third-party computer, bypassing the home agent.

    This approach ensures that the mobile device remains accessible despite moving across networks, and it enables uninterrupted communication.

    • Tunneling: packets are encapsulated and sent between home and foreign agents.

Mobile Computing and Multimedia Applications
  • Transmission Control Protocol (TCP)

    TCP ensures reliable delivery of data packets by retransmitting lost or faulty ones using confirmation messages, ensuring all data arrives correctly. It was developed and optimized for wired networks, where packet loss is rare. However, in mobile networks, TCP faces challenges due to higher error rates and frequent IP address changes.

    • Challenges in Mobile Networks
      • High Error Rates: TCP's reliance on retransmitting all subsequent packets after an error (GoBack-N ARQ protocol) leads to frequent resets, significantly reducing data throughput. This is especially problematic for multimedia applications like video streaming, where interruptions disrupt the user experience.
      • IP Address Dependency: TCP connections depend on TCP ports and fixed IP addresses. When a mobile device switches networks and its IP address changes, the TCP connection is terminated, breaking the corresponding data streams.
    • Improvements to TCP for Mobile Use
      1. Mobile IP Protocol: Maintains a consistent IP address for mobile devices regardless of network switching, preventing connection loss due to IP changes.
      1. Selective Acknowledgment (SACK) protocol: An ARQ variant that retransmits only lost packets instead of all packets after an error, improving performance in error-prone mobile environments.
      1. User Datagram Protocol (UDP): Used for multimedia applications where occasional packet loss is acceptable. UDP avoids delays caused by retransmissions, ensuring smooth playback for video and audio streams. Dropped packets temporarily reduce quality without stalling the stream. Ideal for real-time applications like video or audio streaming, where continuity matters more than perfect quality.
  • Real-Time Streaming Protocol (RTSP)

    RTSP sets up and monitors multimedia streams and supports both TCP (for stable, wired networks) and UDP (for error-prone, mobile networks) for actual media transport. RTSP also supports functions for pausing, scrubbing back and forth, etc.

    • Commands Included
      • OPTIONS: Queries available RTSP commands on the server.
      • DESCRIBE: Retrieves media parameters for specific content.
      • SETUP: Establishes a logical connection to the server.
      • PLAY: Starts playback at a specific position.
      • PAUSE: Temporarily stops the media stream.
      • TEARDOWN: Ends the connection and releases resources.
    • Supporting Protocols

      RTSP is supported by:

      • Real-Time Control Protocol (RTCP): Collaborates with RTP to negotiate an optimal transmission quality and rates. It also provides feedback on the media stream.
      • Real-Time Transport Protocol (RTP): Manages the transmission of data packets, ensuring continuous and adapted media streaming, often using UDP for its connectionless nature.

  • Multimedia Content Adaption

    In addition to the choice of a suitable communication protocol, efficient multimedia streaming on mobile devices requires adjusting content quality to match available bandwidth (transmission rate). This can be achieved through:

    • Pre-Adapting Data: Servers store multiple versions of media content at different quality levels to suit varying transmission rates.
    • Source Encoding: Dynamically compressing high-quality content to lower-quality formats during streaming to suit limited bandwidth. This approach reduces quality (lossy compression) but ensures uninterrupted playback.

6.4. Security and Data Protection in Mobile Systems

Mobile devices require data protection and security against unauthorized access, data theft, and data manipulation. Their mobility makes them more vulnerable due to exposure to unsecured environments and the risk of theft. To combat these threats, implementing strong encryption of the data on the device and rigorous access controls for using the device itself is essential, using specific techniques.

  • Full Disck Encryption (FDE)

    Full Disk Encryption ensures that all data on a device is stored in an encrypted format directly on the hard disk. This process is transparent for the user, who doesn't need to take any extra steps, but it safeguards sensitive data in case the device is stolen because access to the encrypted data is only possible with the correct decryption key. To enhance security, strong passwords or biometric methods (e.g., fingerprint scanners) can be used, making it harder for unauthorized users to access sensitive information.

    This combination of encryption methods and secure protocols provides comprehensive protection for both stored and transmitted data.

    Network Security Considerations:

    Mobile devices often connect to insecure environments like public Wi-Fi, where data can be intercepted by third parties. Encryption of communicated data is crucial in these situations.

    • For private wireless networks, secure encryption protocols like Wi-Fi Protected Access 2 (WPA2) help prevent unauthorized access.
    • In public networks, applications must use secure transport protocols such as Transport Layer Security (TLS) to protect transmitted data. Examples include:
      • HTTPS for web browsers.
      • Secure email protocols like SMTPS, POP3S, or IMAPS to ensure safe communication in email clients.

  • Mobile Virtual Private Networks (VPN)

    A VPN is a subnet that creates a secure, private communication channel over the internet by encrypting data packets exchanged in the VPN. This process, called tunneling, ensures that only authorized participants can access the encrypted content while keeping network traffic hidden from third parties. Key features of a VPN are:

    1. Secure Communication: VPNs provide tap-proof and tamper-proof communication netween participants by encrypting IP packets. These packets are encapsulated within standard internet packets, allowing them to traverse the internet securely, but only the authorized participants of the VPN can read the decrypted contents of the network traffic.
    1. Virtual Subnet: VPNs connect distributed devices and networks over the internet, functioning as a single virtual subnet. Devices in the VPN can use private IP addresses or globally valid IP addresses, depending on the setup. Unlike physical subnets tied to specific locations, VPNs provide global connectivity while maintaining local network protections. Therefore, a VPN simultaneously offers a local network both protection and the global distribution of legitimate participants
    1. Central Role of the VPN Gateway: The VPN gateway manages entry into the virtual subnet, routing traffic securely and facilitating encrypted communication. Common tunneling and encryption protocols include OpenVPN, Internet Key Exchange in version 2 (IKEv2), and Layer 2 Tunneling Protocol (L2TP) along with the IPSec protocol suite.
    • Types of VPNs

      Site-to-Site VPN:

      Connects two or more physical networks (e.g., offices of the same company in different locations) over the internet, creating a unified, secure locally united subnet. Encrypted tunnels ensure all communication between the networks is secure and inaccessible to third parties.

      End-to-End VPN:

      Secures communication between two or more individual devices directly over the internet. Each device uses a private address within the VPN. A VPN gateway acts as a server, providing a secure entry point for encrypted communication between the devices, without requiring a physical subnet. The VPN gateway acts as the entry point for clients to connect securely.

      End-to-Site VPN:

      Allows authorized remote users to securely access a physical subnet’s resources as if they were on-site. It is ideal for remote work scenarios where employees need access to company services from home or the field. It allows access to all company services transparently as if the client were on-site. Remote devices connect to the network using standard VPN solutions as long as the device’s IP address does not change.

      Mobile VPN:

      Provides uninterrupted VPN connectivity for mobile devices that frequently change networks or IP addresses. It is designed for mobile users who experience fluctuating connectivity or frequent network switching.

      A communication server acts as a proxy for the mobile device within the wired network. If the device’s connection is interrupted or its IP address changes, the server maintains the VPN connection by assigning a virtual address among the rage of addresses it manages to the client. This virtual address ensures consistent communication, even as the mobile device reconnects to the network, even if the IP address has changed.This way, the communication server always knows the client’s current address.
